# OOP Data Abstraction via Pure Virtual Functions

This module models pure Data Abstraction by creating strict structural templates (Interfaces) using abstract classes.

## 🧠 Logic & Abstract Interfaces
- **Pure Virtual Syntax (`= 0`):** Forcing a virtual method to equal zero turns the class into an Abstract Class. The system cannot create direct object instances of this parent block.
- **Contract Enforcement:** Any child class that inherits from this abstract parent *must* override the `bootDevice()` function, or the compiler will block execution.

## 🛠️ Concepts Covered
- Pure Virtual Functions
- Data Abstraction & Interface Contracts
- Non-Instantiable Class Architecture
