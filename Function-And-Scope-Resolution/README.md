# Functions, Memory Scopes, and Argument Passing Mechanisms

This module breaks down functional programming by comparing memory allocation during value copies versus memory aliasing.

## 🧠 Logic & Algorithm
- **Call by Value:** Creates a duplicate copy of the variable in a temporary stack frame. Changes made inside the function die when the function scope closes.
- **Call by Reference (`&`):** Passes the actual memory address (alias) of the caller's argument. Any calculation directly overrides the source variable inside `main()`.
- **Global Scope:** Demonstrates data accessible across all functional scopes globally.

## 🛠️ Concepts Covered
- Function Prototyping & Definition
- Local vs Global Scope Resolution
- Stack Memory Duplication (Call by Value)
- Memory Aliasing (Call by Reference)
