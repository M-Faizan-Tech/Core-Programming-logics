# OOP Class Basics and Access Specifiers

Inspired by foundational lab layouts, this project demonstrates data security using access boundaries inside C++ classes.

## 🧠 Logic & Access Controls
- **Private Scope:** Variables `privateX` and `privateY` are restricted. They cannot be altered directly from `main()`, preventing accidental structural shifts.
- **Public Gateways (Getters/Setters):** Public member functions act as secure intermediate channels to input and extract values safely.
- **Public Scope:** Variable `publicZ` is unprotected and open for direct execution anywhere in the script.

## 🛠️ Concepts Covered
- Object-Oriented Blueprinting
- Access Enforcements (`private` vs `public`)
- Safe Encapsulation Routing
